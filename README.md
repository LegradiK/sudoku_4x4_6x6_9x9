# Sudoku

A desktop Sudoku game built with Python and Pygame, supporting 4x4, 6x6, and 9x9 grids with three difficulty levels. Puzzles are generated dynamically via a locally running Docker API.

---

## Features

- Three grid sizes: 4x4, 6x6, and 9x9
- Three difficulty levels: Easy, Medium, Hard
- Real-time wrong cell highlighting in red
- Submit to check your answer (via button or Enter key)
- Restart to reset the current puzzle
- New Game to fetch a fresh puzzle
- How to Play screen on startup
- Keyboard and mouse support

---

## Dependencies

### Python
- Python 3.10+
- [Pygame](https://www.pygame.org/) — game rendering
- [Requests](https://requests.readthedocs.io/) — HTTP calls to the puzzle API

Install Python dependencies:
```bash
pip install pygame requests
```

### Docker
The puzzle generator runs as a local Docker container. Make sure [Docker](https://docs.docker.com/get-docker/) is installed.

## Setup

**1. Clone the repository**
```bash
git clone https://github.com/LegradiK/sudoku_4x4_6x6_9x9.git
cd sudoku_4x4_6x6_9x9
```

**2. (Optional) Create a virtual environment**
```bash
python -m venv venv
source venv/bin/activate  # Windows: venv\Scripts\activate
```

**3. Install Python dependencies**
```bash
pip install pygame requests
```

**4. Start the puzzle generator API**
```bash
docker run -d -p 8080:80 --restart unless-stopped jotools/sudoku
```

> The `--restart unless-stopped` flag means the container starts automatically on every boot — you only need to run this once.

**5. Run the game**
```bash
python main.py
```

---

## How to Run (after first setup)

If the Docker container is already running, just:
```bash
python main.py
```

If it's not running (e.g. after a reboot without `--restart`):
```bash
docker start <container_name>   # find name with: docker ps -a
python main.py
```

## Controls

| Action | Input |
|---|---|
| Move selection | Arrow keys or mouse click |
| Enter a number | 1–x on keyboard or numpad |  *number x to be changed according to the puzzle size
| Erase a cell | Backspace / Delete / `e` / `d` |
| Submit answer | Enter or Submit button |
| New puzzle | New Game button |
| Reset current puzzle | Restart button |

---

## How It Works

Puzzles are generated by the [`jotools/sudoku`](https://hub.docker.com/r/jotools/sudoku) Docker image, which exposes a REST API at `localhost:8080`. The game sends a GET request on each New Game:

```
GET /api/sudoku/generate?size=9&numClues=30&format=json&addSolution=true
```

The response contains the puzzle cells and the solution, which the game uses to validate answers.
